# KSC Pilot — Coach Quick Start

## Purpose

This quick-start guide is for KSC pilot coaches using SIC during Week 20 pilot readiness and the first real coach pilot sessions.

It is designed to help a coach get from sign-in to a usable session as quickly as possible.

## What SIC is for in this pilot

In this pilot, SIC helps you:

- create a training session from real coaching constraints
- use Fut-Soccer-biased defaults where configured
- use image-assisted intake where available
- review and adjust the generated result
- save and export your session
- submit practical feedback after use

This is a focused pilot workflow, not a full club platform.

## Before you start

Make sure you have:

- the approved KSC pilot login entry path: `/login`
- your approved coach access
- a clear idea of your session needs, such as:
  - flow choice such as Soccer or Fut-Soccer where relevant
  - age band
  - session duration
  - available equipment
  - session theme
- an environment or setup photo if you plan to use the image-assisted flow

If you expected access and cannot sign in, contact the pilot operator.

## Step 1 — Sign in

1. Open `/login`.
2. Select **Sign in**.
3. Complete the existing approved sign-in flow.
4. Continue into the SIC coach app at `/sessions/new`.

You do not need to:

- enter a tenant ID
- choose an organization
- choose a role
- use a technical login path not provided by the operator

## Step 2 — Start a session

After sign-in, you should land on `/sessions/new`.

Start a new session using the current Session Builder path.

You may be asked for or may choose to provide details such as:

- flow choice such as Soccer or Fut-Soccer
- age band
- duration in minutes
- session theme
- available equipment
- an optional confirmed image-analysis profile if you use the image-assisted path

Be practical and specific. The better your inputs, the more useful the result is likely to be.

## Step 3 — Use image-assisted intake if needed

If the image-assisted flow is enabled for your pilot use, you may use it in one of two ways.

### Option A — Environment photo

Use an image to help SIC understand your training environment before generating a session.

Use this when you want SIC to understand the space or setup you are working with.

### Option B — Setup-to-drill photo

Use an image of a setup to help SIC generate a drill suggestion from what you already arranged.

Use this when you want help turning a real setup into a drill idea.

Important:
Always review and confirm image-derived results before using them. SIC is meant to assist your coaching judgment, not replace it.

## Step 4 — Review the generated result

Once SIC generates a result, review it carefully.

Look for:

- whether the session matches the duration you requested
- whether the activities fit the equipment you provided
- whether the session theme is correct
- whether the age band and overall difficulty feel right
- whether any image-derived interpretation needs correction

If something is off, adjust the flow using the available current editing or retry path.

## Step 5 — Save your work

When the session looks usable, save it through the current SIC flow.

Save once you are comfortable that:

- the session is close enough to run
- the coaching objective is clear
- the activities are practical for your setup

If save does not behave as expected, note where the issue happened and contact the pilot operator.

## Step 6 — Export your session

If export is available in your current flow, export the session after review or save.

Use export when you want a version you can take into training or share for your own coaching use.

Before export, make sure the session content is ready enough to use in practice.

## Step 7 — Submit feedback

After using SIC, submit pilot feedback through the current workflow where available.

The pilot is specifically looking for feedback on:

- session quality
- drill usefulness
- image analysis accuracy
- missing features

Useful feedback is short and practical.

Examples:

- “The session fit our numbers and space well.”
- “The drill idea was useful but needed easier progressions.”
- “The image analysis misunderstood the setup shape.”
- “I wanted a faster way to edit the final activity.”

## What to do if something fails

If you run into a problem:

1. note where it happened
2. retry only if it seems like a temporary issue
3. contact the pilot operator if you are blocked

Helpful issue details include:

- sign-in problem
- session generation problem
- image-assisted problem
- save problem
- export problem
- feedback submission problem

Also include what you expected and what happened instead.

## Best practices for this pilot

To get the best results:

- use real coaching constraints
- keep your request specific
- review generated content before using it
- confirm image-assisted output carefully
- save useful sessions
- submit feedback even when the session mostly worked

The pilot is most valuable when feedback comes from real coaching use, not only perfect happy-path use.

## What not to expect in this pilot

This pilot does not yet aim to provide:

- a full club operating system
- every advanced team workflow
- perfect automation for every coaching need
- broad analytics or reporting
- unlimited customization across every use case

This is a focused pilot around the current shared Session Builder flow.

## Quick checklist

Before training, confirm:

- I can sign in through the approved pilot path
- I can start a session
- I reviewed the generated output
- I confirmed any image-assisted result
- I saved or exported what I need
- I know where to leave feedback

## Support

If you expected pilot access but cannot continue, contact the KSC pilot operator.

If something in the product flow fails, report:

- what step you were on
- what you expected
- what happened
- whether you were using the standard or image-assisted flow

## Definition of success

This quick-start guide succeeds if a coach can:

- get into SIC
- create a session
- use the image-assisted flow where relevant
- review, save, and export
- leave practical feedback
without needing deep technical explanation
